Vec3.hpp — vektor a operátory¶

Nejjednodušší soubor: hodnotový typ pro 3D vektor. Ukazuje přetěžování operátorů, inline funkce a agregátní inicializaci.

Celý soubor¶

#pragma once

struct Vec3 {
    double x = 0.0, y = 0.0, z = 0.0;
    Vec3() = default;
    Vec3(double a, double b, double c) : x(a), y(b), z(c) {}
};

inline Vec3  operator+(Vec3 a, Vec3 b){ return {a.x+b.x, a.y+b.y, a.z+b.z}; }
inline Vec3  operator-(Vec3 a, Vec3 b){ return {a.x-b.x, a.y-b.y, a.z-b.z}; }
inline Vec3  operator*(double s, Vec3 a){ return {s*a.x, s*a.y, s*a.z}; }
inline Vec3& operator+=(Vec3& a, Vec3 b){ a.x+=b.x; a.y+=b.y; a.z+=b.z; return a; }

inline double mag2(double a){ return a*a; }
inline double mag2(Vec3 a){ return a.x*a.x + a.y*a.y + a.z*a.z; }

Jazykové prvky¶

#pragma once¶

Zajišťuje, že se hlavička při vícenásobném #include vloží jen jednou (náhrada za klasické include guards #ifndef/#define). Nestandardní, ale podporované všemi běžnými překladači.

struct s výchozími hodnotami členů¶

double x = 0.0, y = 0.0, z = 0.0;

C++11 default member initializer — nový Vec3 má složky vynulované. Ve struct jsou členy public (na rozdíl od class), což je pro prostý datový typ žádoucí.

Dva konstruktory¶

Vec3() = default;                          // prázdný → {0,0,0}
Vec3(double a, double b, double c) : ...    // ze tří složek

= default říká překladači „vygeneruj výchozí konstruktor". Druhý konstruktor používá inicializační seznam (: x(a), y(b), z(c)) — inicializuje členy přímo, ne přiřazením v těle (rychlejší, nutné pro const/reference členy).

Přetěžování operátorů¶

inline Vec3 operator+(Vec3 a, Vec3 b){ return {a.x+b.x, ...}; }

Umožní psát U[c] - Uprev[c] místo subtract(U[c], Uprev[c]). Operátory jsou volné funkce (ne členské), což je pro symetrické binární operace idiomatické.

  • operator* je definován jako double * Vec3 (skalár vlevo) — používá se ve výrazech typu rAU[c] * Hf[c].
  • operator+= vrací referenci Vec3& a bere první argument referencí, aby šlo řetězit a modifikovat původní objekt.

inline¶

Tyto krátké funkce jsou v hlavičce a označené inline, aby je šlo vložit do více překladových jednotek bez chyby „multiple definition" a aby je překladač mohl vložit přímo do volajícího kódu (eliminace režie volání).

Návrat {...} — agregátní inicializace¶

return {a.x+b.x, a.y+b.y, a.z+b.z};

Složené závorky vytvoří Vec3 přímo z hodnot — překladač ví, že návratový typ je Vec3. Stručné a bez explicitního konstruktoru.

Přetížení mag2 podle typu¶

inline double mag2(double a){ return a*a; }
inline double mag2(Vec3 a){ return a.x*a.x + ...; }

Dvě funkce stejného jména, různé argumenty — function overloading. Klíčové pro fvMatrix: tamní šablonový kód volá mag2(np - psi[c]) a podle toho, zda je T = double nebo Vec3, se vybere správná verze. Tím funguje výpočet rezidua jednotně pro tlak i rychlost.

Proč hodnotový typ, ne pole [3]

Vec3 jako struct se třemi double se předává hodnotou (24 bytů), překladač ho drží v registrech a inlinuje operace. Pojmenované složky .x .y .z jsou čitelnější než [0][1][2] a typový systém zabrání záměně skaláru a vektoru.
